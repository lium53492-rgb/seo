## Objective and cadence

Build a small, honest topic cluster around story-led AI voice roleplay. The cluster must answer different user questions rather than publish keyword variants of the same page.

- 09:15 Asia/Shanghai: primary page.
- 16:15 Asia/Shanghai: a temporary companion page only when it has a distinct intent, new evidence, a safe fact-based brief, and a useful internal-link role. This is the explicitly approved one-week exception to the usual one-page-per-day limit.
- The 16:15 run appends to the same daily research file, report, and PDF; it does not create a separate daily report.
- Every entry is a planning candidate, not a publication promise. The daily research run may replace or skip it when public evidence, product-fit, duplicate-content, build, or production checks fail.

## Decision rules for every planned page

1. Re-score the candidate from fresh public evidence on its actual day. Store only transparent `demandScore` and `difficulty` proxy scores; never represent them as search volume, CPC, Semrush KD, or Google data.
2. Compare title, H1, first-screen question, headings, and internal-link role against all existing pages. Replace the candidate if it cannot state a distinct reader question in one sentence.
3. Use only the approved facts: story-led AI voice roleplay, an existing plot rather than an empty chat, selection from available story characters, and the team’s interactive-fiction history.
4. Exclude celebrity, film, television, anime, sports-event, game, and other third-party IP. A trend can be a lead but never a reason to publish unsupported or infringing copy.
5. Publish only after the research builder, content quality gates, `npm.cmd run check`, `npm.cmd run build`, production deployment, rendered H1, CTA destination, and sitemap all pass.
6. If an afternoon page fails any gate, do not substitute a rushed page. Record the reason in the shared daily report and let the next morning’s primary page proceed normally.

## Measurement review

- The morning run reads Search Console’s prior 28 days when the signed-in property is available; its clicks, impressions, CTR, and positions stay separate from planning proxies.
- At the 16:15 run, update the same report/PDF with the day’s second-page decision and any new performance data.
- Review each URL at 7, 14, and 28 days for impressions first, then query match, CTR/title fit, and clicks. Do not promise 20 UV per page: Search Console clicks and Vercel visitors are different measures, and new URLs need indexing evidence before conversion conclusions.
